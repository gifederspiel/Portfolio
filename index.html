Story Poker S/4HANA · SAPUI5 · WebSockets

A real-time planning-poker tool for sprint refinements: teams join a shared room and vote on the complexity of a ticket in story points, so estimates stay live and in sync across everyone in the session.

What it does
Create a room per team, configure which values can be voted on, and run live estimation rounds. The tool aggregates each round with simple calculations to suggest the best value, and shows how every member voted — or hides it via anonymous voting, which is used for the end-of-sprint confidence vote.
My Role
Designed and built the whole tool on my own, from the room model and configuration UI to the real-time voting flow, deployed for internal company use.
What I learned
Working with WebSockets to keep every participant's view in sync in real time, and modelling multi-team rooms with configurable voting options.
