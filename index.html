04 — Projects

Selected Projects

HomeBite

React Native · Expo · Supabase · Claude AI

A cross-platform food-inventory app that cuts down on food waste: track what's in your kitchen and its expiry dates, then get AI-generated meal suggestions based on what you already have.

What it does
Expiry tracking with AI meal suggestions, fast item entry via receipt scanning and voice, and sign-in with email, Apple or Google.
How it's built
Expo / React Native app on a Supabase backend (auth, Postgres, edge functions), Claude proxied through an edge function, and a Liquid-Glass UI built with expo-blur.
What I learned
Wiring an LLM into a real product end to end, integrating camera and voice input, and shipping a tested mobile app with EAS and CI.
GitHub Repo →

Strava Dashboard

Node.js · Express · Vite · Strava API

A personal dashboard that pulls in my own Strava data and shows weekly and monthly training stats, route maps and workout details in one place.

Why I built it
Wanted a cleaner overview of my training than the Strava app gives, and an excuse to work the Strava API end to end.
What I learned
OAuth flows and token refresh, building a small Node/Express proxy to keep secrets off the client, and wiring it together with a Vite frontend.
Live App → GitHub Repo →

Story Poker

S/4HANA · SAPUI5 · WebSockets

A real-time planning-poker tool for sprint refinements: teams join a shared room and vote on ticket complexity in story points, keeping estimates live and in sync across the session.

What it does
Create a room per team, configure votable values, and run live estimation rounds. It aggregates each round to suggest a value and shows how everyone voted — or hides it for anonymous confidence votes.
My role
Designed and built the whole tool solo, from the room model and configuration UI to the real-time voting flow, deployed for internal company use.
What I learned
Using WebSockets to keep every participant's view in sync in real time, and modelling multi-team rooms with configurable voting options.
